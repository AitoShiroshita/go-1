Goのソースコードをビルドしたり処理するプログラム一式はあります。 これらは直接起動するよりも、goプログラムによって起動されることがほとんどです。

これらのプログラムを起動する一般的な方法は、例えばgo fmtのように、goプログラムのサブコマンドとしてです。 このように起動すると、コマンドはGoのソースコードのパッケージ群全体に対して、パッケージごとに基となるバイナリを引数付きで起動し、適切に処理されます。

プログラムは単独のバイナリとしても起動できます。 go toolサブコマンドを使うことで、例えばgo tool vetのように、引数を変えずに起動できます。 この実行スタイルを使うことで、例えばパッケージ全体ではなく、一つのソースファイルをチェックすることができます: go vet mypackageに対して、go tool vet myprogram.go。 いくつかのコマンド、例えばyaccなどは、go toolサブコマンドからのみ使用可能です。

最後に、fmtとgodocコマンドはよく使用されるため、 gofmtとgodocという普通のバイナリとしてインストールされています。

より詳細なドキュメントや起動方法、使用方法をみるには、リンクをクリックしてください。

名前      概要
go      goプログラムはGoのソースコードを処理し、ここに列挙しているコマンドを実行します。 使い方の詳細は、コマンドのドキュメントをみてください。

cgo      CgoはCのコードを呼び出すGoパッケージの作成を可能にします。
cover      Coverは"go test -coverprofile"によって生成されるカバレッジプロファイルを解析するプログラムです。
fix      Fixは古い言語機能やライブラリを見つけ、新しいものに書き換えるGoプログラムです。
fmt      FmtはGoパッケージのフォーマットを整えます。 これは独立したgofmtコマンドとしても使用でき、より汎用的なオプションを指定できます。
godoc      GodocはGoパッケージのドキュメントを抽出し、生成します。
vet      VetはGoのソースコードを調査し、怪しい構文、例えば引数がフォーマット文字列と揃っていないPrintf呼び出しを報告します。
yacc      YaccはGoで実装されたパーサーを出力するバージョンのyaccです。

このリストは要約版です。 コンパイラなどのドキュメントは、全コマンドのリファレンスをみてください。
